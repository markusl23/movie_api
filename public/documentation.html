Movie-API – Documentation

A simple RESTful backend for movie information and user favorites

1. Overview

The Movie-API is a server-side application that provides information about movies, their directors and genres. In addition, users can create a personal account and maintain a list of their favorite movies.

The API is designed as a REST service and can be consumed by different clients (for example a single-page application, a mobile app, or other services).

2. Main Resources

Movies

Movies are the central resource of the API. For each movie, the backend stores at least:

  • Title
  • Description
  • Genre
  • Director
  • Image URL (poster)
  • Whether the movie is featured or not

Genres

Genres describe the type of movie (for example Thriller, Drama or Comedy) and include at least a short textual description.

Directors

For each director the API stores biographical information, such as:

  • Name
  • Bio
  • Year of birth
  • Year of death (if applicable)

Users

Users can register with the Movie-API in order to maintain their own profile and a list of favorite movies. A user record contains for example:

  • Username
  • Password (stored securely)
  • Email
  • Date of birth
  • List of favorite movies

3. Current API Endpoints

The following table shows the REST endpoints that the Movie-API exposes.

Request URL HTTP Method Request body data format Response body data format
Get all movie data /movies GET none Array holding JSON objects with movie data
Get all user data /users GET none Array holding JSON objects with user data
Get all genre data /genres GET none Array holding JSON objects with genre data
Get all director data /directors GET none Array holding JSON objects with director data
Get single movie data /movies/:name GET none JSON object holding single movie data
Get single genre data /genres/:name GET none JSON object holding single genre data
Get single director data /directors/:name GET none JSON object holding single director data
Add data for new user /users POST JSON object containing user data, example: {"name": "Michael","email": "michael@email.net","favorites": ["Gladiator", "Point Break"]} error message or new user data including v4 uuid, example: {"name":"Michael","email":"michael@email.net","favorites": ["Gladiator","Point Break"],"id":"1abb2ef7-1e14-4dfb-bcd9-48bc5442d05f"}
Get single user by ID /users/:id GET none error message or JSON object holding single user data
Delete single user by ID /users/:id DEL none error message or success message containing former user ID
Update user name /users/:id/user_name/:name PUT none error message or success message stating the user name change
Update user email /users/:id/user_email/:email PUT none error message or success message stating the user email change
Add movie to user favorites /users/:id/favorites/:title PUT none Only shows placeholder message, handler needs to be set up
Remove movie from user favorites /users/:id/favorites/:title DEL none Only shows placeholder message, handler needs to be set up

4. Authentication & Security

The Movie-API is intended to be protected by user authentication. Only registered users should be able to manage their profile and favorite movies. Typical implementations use token-based authentication (JSON Web Tokens, JWT) and validate all incoming data to keep the application and the stored user information secure.

5. Usage

A frontend application can use the Movie-API by sending HTTP requests to the endpoints listed above. The API responds with JSON objects that contain the requested data or confirmation messages for successful operations (for example after updating user information or changing the list of favorite movies).

This documentation page is meant as a simple overview for developers who want to understand what the Movie-API does and which resources it exposes.
