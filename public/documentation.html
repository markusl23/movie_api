Movie-API – Documentation

A simple RESTful backend for movie information and user favorites

1. Overview

The Movie-API is a server-side application that provides information about movies, their directors and genres. In addition, users can create a personal account and maintain a list of their favorite movies.

The API is designed as a REST service and can be consumed by different clients (for example a single-page application, a mobile app, or other services).

2. Main Resources

Movies

Movies are the central resource of the API. For each movie, the backend stores at least:

  • Title
  • Description
  • Genre
  • Director
  • Image URL (poster)
  • Whether the movie is featured or not

Genres

Genres describe the type of movie (for example Thriller, Drama or Comedy) and include at least a short textual description.

Directors

For each director the API stores biographical information, such as:

  • Name
  • Bio
  • Year of birth
  • Year of death (if applicable)

Users

Users can register with the Movie-API in order to maintain their own profile and a list of favorite movies. A user record contains for example:

  • Username
  • Password (stored securely)
  • Email
  • Date of birth
  • List of favorite movies

3. Example Endpoints

The following list shows typical REST endpoints that the Movie-API exposes. These URLs are examples and can be adjusted to the final implementation.

GET /movies – return a list of all movies.
GET /movies/:title – return details about a single movie by its title.
GET /genres/:name – return information about a specific genre.
GET /directors/:name – return biographical information about a director.
POST /users – register a new user.
PUT /users/:username – update an existing user’s profile data.
POST /users/:username/favorites/:movieId – add a movie to the user’s favorites.
DELETE /users/:username/favorites/:movieId – remove a movie from the favorites list.
DELETE /users/:username – delete an existing user account.

4. Authentication & Security

The Movie-API is intended to be protected by user authentication. Only registered users should be able to manage their profile and favorite movies. Typical implementations use token-based authentication (JSON Web Tokens, JWT) and validate all incoming data to keep the application and the stored user information secure.

5. Usage

A frontend application can use the Movie-API by sending HTTP requests to the endpoints listed above. The API responds with JSON objects that contain the requested data or confirmation messages for successful operations (for example after updating user information or changing the list of favorite movies).

This documentation page is meant as a simple overview for developers who want to understand what the Movie-API does and which resources it exposes.
